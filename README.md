# Sand Mine Distance Finder — Hosting & Data Update Guide

## One-time setup (~10 minutes)

### Step 1 — GitHub
1. Go to **github.com** and create a free account (or sign in)
2. Click **New repository** → name it `sand-finder` → set to **Public** → click **Create repository**
3. Click **Add file → Upload files**
4. Upload everything in this folder: `index.html`, `update_data.py`, and the entire `data/` folder
5. Click **Commit changes**

### Step 2 — Netlify
1. Go to **netlify.com** → Sign up free (use your GitHub account to sign in — easiest)
2. Click **Add new site → Import an existing project → GitHub**
3. Select your `sand-finder` repository
4. Leave all build settings blank (no build command needed)
5. Click **Deploy site**
6. In ~30 seconds you'll have a live URL like `sparkly-muffin-abc123.netlify.app`
7. Optional: in **Site settings → General → Site name** change it to something like `sand-finder-tx` for a cleaner URL

**That's it.** Open the URL on your phone, add it to your home screen (Safari → Share → Add to Home Screen), and it works like an app.

## Updating data

When you have fresh CSVs from your data source:

1. **Replace** the three CSV files in this folder with the new ones (keep the same filenames)
2. **Run the update script:**
   ```
   python update_data.py
   ```
   This overwrites the four files in `data/`
3. **Push to GitHub** — two options:
   - **Option A (easiest):** Go to your GitHub repo in a browser → open the `data/` folder → drag and drop the four new JSON files → commit
   - **Option B (faster once set up):** Use GitHub Desktop (free app) to sync the folder
4. **Netlify auto-deploys** within 30 seconds of your GitHub push — no action needed

## File structure
```
sand-finder/
├── index.html          ← The app (never needs to change)
├── update_data.py      ← Run this to rebuild data files from CSVs
├── README.md           ← This file
└── data/
    ├── wells.json      ← Generated by update_data.py
    ├── permits.json    ← Generated by update_data.py
    ├── rigs.json       ← Generated by update_data.py
    └── meta.json       ← Generated by update_data.py (operator/state lists)
```

## Load sizes (what your phone actually downloads)
| File | Raw | Gzipped (actual download) |
|------|-----|--------------------------|
| index.html | 39 KB | ~12 KB |
| wells.json | 3.4 MB | 476 KB |
| permits.json | 1.6 MB | 328 KB |
| rigs.json | 79 KB | 16 KB |
| meta.json | 24 KB | 8 KB |
| **Total** | **~5 MB** | **~840 KB** |

Netlify serves all files with gzip compression automatically — so the actual data your phone downloads is about 840 KB total, which loads in 1–2 seconds on LTE.

## Requirements for update script
- Python 3.7+
- pandas (`pip install pandas`)
